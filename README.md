## ◈ Featured Projects

<details>
<summary><b>◈ Honeypot Tracker & Log Analyzer</b> &nbsp;|&nbsp; Network Security Research Tool</summary>

<br/>

A lightweight network honeypot engineered for reconnaissance pattern analysis in controlled, isolated environments. Built to surface scanning behavior, repeated probes, and common attack vectors through automated log aggregation and triage tooling.

| Attribute | Detail |
|---|---|
| **Stack** | Python, Sockets, VirtualBox |
| **Scope** | Network-layer honeypot with inbound connection logging and payload metadata capture |
| **Performance** | Automated log aggregation reducing initial triage time significantly |
| **Security** | Runs exclusively on isolated VMs; full documentation on safe deployment and privacy |
| **Impact** | Surfaces common scanning ports and repeated probe patterns for threat intelligence |
| **Repository** | [github.com/darplalani08](https://github.com/darplalani08) |

The honeypot captures IP addresses, ports, timestamps, and payload metadata for every inbound connection attempt. Log data is automatically aggregated and analyzed to identify the most commonly targeted ports and repeated adversarial probes. All operations are confined to isolated virtual machine environments, with clear ethical documentation covering deployment boundaries and privacy considerations — designed as a research asset, not an offensive tool.

</details>

<details>
<summary><b>◈ KeyLogger — Defensive Research Module</b> &nbsp;|&nbsp; Endpoint Behavior Analysis</summary>

<br/>

A controlled key-capture module developed strictly for defensive security research and endpoint behavior analysis. Built and tested exclusively within isolated virtual machine environments, this tool is engineered to study input-capture techniques and generate detection indicators for endpoint protection systems.

| Attribute | Detail |
|---|---|
| **Stack** | Python, Linux, VirtualBox |
| **Scope** | Key-capture module for studying persistence and input-interception techniques |
| **Performance** | Parsing scripts extracting meaningful artifacts from raw logs efficiently |
| **Security** | Isolated VM deployment only; documented safe handling and remediation procedures |
| **Impact** | Detection indicators and mitigation notes for endpoint protection teams |
| **Repository** | [github.com/darplalani08](https://github.com/darplalani08) |

The module focuses on identifying common persistence mechanisms and input-capture techniques used by real-world malware, then producing actionable detection indicators for defenders. Parsing scripts extract meaningful artifacts from generated logs. All research outputs include documented safe handling procedures and remediation steps, making this a reference artifact for endpoint protection engineering rather than an exploitation asset.

</details>
