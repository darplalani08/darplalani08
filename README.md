## `> ls ./projects`

<details>
<summary><b>[ PROJECT_01 ] &nbsp; Aegis — Reverse Proxy Web Application Firewall</b> &nbsp;|&nbsp; <code>Active Development</code></summary>

<br/>

**Aegis** is an ongoing Web Application Firewall project built as a reverse proxy layer that sits between the client and backend server. Its core mission: intercept, inspect, and filter malicious traffic in real time — blocking common web-based attacks before they reach application logic.

| Attribute | Detail |
|---|---|
| **Stack** | Python, Sockets, HTTP Parsing, Linux |
| **Scope** | Reverse proxy WAF with real-time payload inspection and rule-based filtering |
| **Focus Areas** | Packet Inspection · SQLi Detection · XSS Mitigation · Traffic Filtering |
| **Security** | All development and testing conducted in isolated, controlled environments |
| **Impact** | Intercepts and mitigates common OWASP Top 10 web attack vectors pre-backend |
| **Status** | 🟡 Active — Under development |
| **Repository** | [github.com/darplalani08](https://github.com/darplalani08) |

Aegis operates as a transparent reverse proxy: all inbound HTTP/HTTPS traffic passes through its inspection engine before being forwarded. The engine evaluates payloads against a signature ruleset covering SQL injection patterns, cross-site scripting vectors, and anomalous request structures. Flagged requests are blocked and logged with full context for forensic review. The project targets the fundamentals of application-layer defense — understanding how attacks are structured, so interception can be precise and low-latency.

</details>

<details>
<summary><b>[ PROJECT_02 ] &nbsp; Honeypot Tracker & Log Analyzer</b> &nbsp;|&nbsp; <code>Network Security Research</code></summary>

<br/>

A lightweight network honeypot engineered for reconnaissance pattern analysis in controlled, isolated environments. Built to surface scanning behavior, repeated probes, and common attack vectors through automated log aggregation and triage tooling.

| Attribute | Detail |
|---|---|
| **Stack** | Python, Sockets, VirtualBox |
| **Scope** | Network-layer honeypot with inbound connection logging and payload metadata capture |
| **Performance** | Automated log aggregation reducing initial triage time significantly |
| **Security** | Runs exclusively on isolated VMs; full documentation on safe deployment and privacy |
| **Impact** | Surfaces common scanning ports and repeated probe patterns for threat intelligence |
| **Repository** | [github.com/darplalani08](https://github.com/darplalani08) |

The honeypot captures IP addresses, ports, timestamps, and payload metadata for every inbound connection attempt. Log data is automatically aggregated and analyzed to surface the most commonly targeted ports and repeated adversarial probes. All operations are confined to isolated virtual machine environments, with clear ethical documentation covering deployment boundaries and privacy considerations.

</details>

<details>
<summary><b>[ PROJECT_03 ] &nbsp; KeyLogger — Defensive Research Module</b> &nbsp;|&nbsp; <code>Endpoint Behavior Analysis</code></summary>

<br/>

A controlled key-capture module developed strictly for defensive security research and endpoint behavior analysis. Built and tested exclusively within isolated virtual machine environments to study input-capture techniques and generate detection indicators for endpoint protection systems.

| Attribute | Detail |
|---|---|
| **Stack** | Python, Linux, VirtualBox |
| **Scope** | Key-capture module for studying persistence and input-interception techniques |
| **Performance** | Parsing scripts extracting meaningful artifacts from raw logs efficiently |
| **Security** | Isolated VM deployment only; documented safe handling and remediation procedures |
| **Impact** | Detection indicators and mitigation notes for endpoint protection teams |
| **Repository** | [github.com/darplalani08](https://github.com/darplalani08) |

The module identifies common persistence mechanisms and input-capture techniques used by real-world malware, then produces actionable detection indicators for defenders. Parsing scripts extract meaningful artifacts from generated logs. All outputs include safe handling procedures and remediation steps — a reference artifact for endpoint protection engineering, not exploitation.
